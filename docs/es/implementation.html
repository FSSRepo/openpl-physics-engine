Implementacion

Open PL puede ser compilado en cualquier plataforma, solo implementalo en tu proyecto C++ o Java y comienza a crear cosas muy increibles.

Implementacion como libreria statica

Esta forma de implementar es incluirlo en el interior del ejecutable, no sera necesario tener un archivo dll complementario.

Implementacion como libreria dinamica

Esta forma de implementar es incluir un archivo dll como complemento del ejecutable, no hay tanta diferencia es solo cuestion decisiva de uno mismo.

Implementacion como codigo fuente

Esta forma de implementar es incluir y poder modificar la API para hacerla mas personalizable y comoda para usar.

Android Studio

Implementacion en Android Studio para desarrollar en la plataforma android.

Binarios
Android arm-v7, arm64, x86, x86_64

Implementacion como codigo nativo

Esta forma de implementarlo consiste en incluir todo el codigo en una sola libreria C++ que podria ser la de tu proyecto si es que tu estas desarrollando de forma nativa una aplicacion que lo requiera.

Implementacion como backend java

Esta forma de implementar es incluir los arhivos de libreria precompilados y una libreria java precompilada para solo poner en las dependencias del proyecto.

Netbeans

Implementacion en el IDE de java para proyectos de desktop.

Binarios
openpl-java-32bits
openpl-java-64bits

La implementacion consiste en crear una libreria para acceder a los metodos nativos de la API.
